The "MAP OF YOUR ADVENTURE" → Bulk SVG Upload (Index) section is not updating correctly. Every time the user uploads a new set of SVG transparent icons, the application continues using the previously mapped default icons instead of the newly uploaded ones.

Implement the following logic:

Completely remove all built-in/default SVG icon mappings from the application. They must never be used again once the user uploads custom SVG icons.
When a new SVG icon set is uploaded:
Delete all previously uploaded SVG icons.
Remove all default icon references.
Clear all caches, memory objects, object URLs, local storage, session storage, IndexedDB entries, and temporary files related to the previous icon set.
Reset the entire icon-mapping state before importing the new files.
Register only the SVG transparent icons uploaded during the current upload session.
Rebuild the category-to-icon mapping using only the newly uploaded SVG files.
Update the upload preview immediately so it displays only the current SVG icons.
Save the current uploaded SVG icon set as the active project icon library.
During PDF generation, the "MAP OF YOUR ADVENTURE" page must use only the currently active uploaded SVG transparent icons.
Never merge old and new icon sets. Each upload must be treated as a complete replacement, not an update.
If the user leaves an upload slot empty, keep that slot empty. Do not reuse an icon from a previous upload or from the application's default icon set.
Ensure that no hardcoded fallback icons are used once the user has uploaded custom SVG icons.
Synchronize the upload panel, icon mapping, preview, and PDF generation so they all reference the same active SVG icon set.
Expected Result

After a new SVG transparent icon set is uploaded:

All previous default and previously uploaded icons are permanently removed from the current project.
Only the newly uploaded SVG transparent icons are visible in the upload panel, preview, and generated PDF.
The "MAP OF YOUR ADVENTURE" page displays only the user's latest uploaded SVG icons with correct mapping, transparent backgrounds, sharp vector quality, and no traces of the old default icon set.

The previous **"MAP OF YOUR ADVENTURE" Bulk SVG Upload (Index)** workflow has been removed. All previously uploaded or hardcoded SVG transparent icons must be permanently discarded.

### 1. Remove Previous Icon System Completely

Completely remove the previous icon system from the application.

This includes:

* All default SVG icon mappings.
* Bulk SVG Upload (Index) logic.
* Upload state.
* Cached SVG files.
* Memory references.
* Local storage/session storage data.
* Object URLs.
* Temporary files.
* Fallback mappings.
* Default initialization logic.

After this cleanup, there must be **no remaining references** to the old icon system anywhere in the application or PDF generation process.

---

### 2. Use the New Default SVG Icon Library

From now on, the application must use the **official SVG icon library** hosted at the provided GitHub URLs.

These SVG files are now the permanent default icons for the **"MAP OF YOUR ADVENTURE"** page.

No user upload is required.

The renderer must automatically retrieve the correct SVG icon based on its **file name**.

### 4. Load Icons from the Official URLs

Each icon must be downloaded from its corresponding GitHub Raw URL at runtime or bundled during the build process.

Requirements:

* Use only the provided Raw GitHub URLs.
* Preserve SVG vector quality.
* Maintain transparency.
* Do not rasterize the SVGs.
* Render them as vector graphics in the final PDF.

---

### 5. Cache Management

Implement proper caching.

* Download each SVG only once.
* Cache the SVG locally after the first successful download.
* If the icon already exists in the cache, reuse it.
* If the cache is invalid or missing, automatically re-download the SVG.
* Never use stale or outdated icon data.

---

* Preserve its original aspect ratio.

## Book Cover Generator – Add Fully Customizable Book Spine Title Editor

### Required Implementation

In the all **Book Cover Generator** sections, add a dedicated **Book Spine Title Editor** that allows the user to customize the text displayed in the **spine area** between the front cover and back cover.

The spine title must be positioned automatically within the printable spine area based on the selected KDP cover dimensions and page count.

---

## Enable / Disable Spine Title Editor

Add a toggle option:

**Book Spine Title Editor**

* **Enable**
* **Disable**

### Logic

* **Disabled (Default):**

  * No spine title should be displayed.
  * All spine title editing controls should remain hidden or disabled.
  * The spine area should remain blank.

* **Enabled:**

  * Display the spine title on the cover preview.
  * Enable all spine title customization options.
  * Apply all user changes instantly in the live preview and final PDF.

The user should be able to turn the spine title on or off at any time without affecting the rest of the cover design.

---

## Spine Title Input

Provide a text input where the user can enter the book title that will appear on the spine.

Example:

* Book Title
* Series Name
* Volume Number
* Author Name (optional)

The spine text should update in real time on the cover preview.

---

## Spine Text Customization Options

Allow the user to fully customize the spine title with the following options:

* Font family
* Font size
* Bold
* Italic
* Underline (optional)
* Text color
* Background color (or transparent)
* Letter spacing
* Line spacing (if multiple lines are supported)
* Text alignment (center, top, bottom within the spine)
* Text rotation (vertical or horizontal, depending on spine width)

---

## Automatic Spine Size Logic

Implement responsive sizing based on the calculated spine width.

The application must automatically calculate the available spine width using the selected trim size, page count, paper type, and KDP specifications.

Then:

* If the spine is wide, allow a larger font size.
* If the spine is narrow, automatically reduce the font size so the title fits completely within the spine.
* Prevent text overflow, clipping, or truncation.
* Ensure the title always remains centered and fully visible.

The font size should adjust dynamically while preserving readability.

---

## Layout Rules

The spine title must:

* Stay completely inside the spine area.
* Never overlap the front cover or back cover.
* Never extend into the bleed or safe margins.
* Remain perfectly centered within the spine.
* Scale automatically whenever the user changes the book size or page count.

---

## Live Preview

Whenever the user changes:

* Book Spine Title Editor (Enable/Disable)
* Book title
* Font
* Font size
* Text color
* Background color
* Bold
* Italic
* Book dimensions
* Page count

the cover preview should update instantly without requiring regeneration.

---

## PDF Export Requirements

The generated print-ready KDP cover PDF must:

* Preserve the exact spine title formatting shown in the preview.
* Embed fonts correctly.
* Keep the spine text sharp and high quality.
* Maintain correct alignment and spacing.
* Export the spine title only when the **Book Spine Title Editor** is enabled.
* Export a blank spine (without title) when the **Book Spine Title Editor** is disabled.

---

## Validation Checklist

Before generating the final cover PDF, verify that:

* ✅ The user can enable or disable the **Book Spine Title Editor**.
* ✅ The user can enter a custom spine title.
* ✅ The spine title is positioned only within the spine area.
* ✅ Font, size, bold, italic, text color, and background color are fully customizable.
* ✅ The font size adjusts automatically based on the calculated spine width.
* ✅ The title never overlaps the front or back cover.
* ✅ The title remains centered and fully visible.
* ✅ The preview matches the exported PDF exactly.
* ✅ The spine title is included only when enabled and omitted when disabled.
* ✅ The final cover complies with Amazon KDP spine layout requirements.

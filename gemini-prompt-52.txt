The bullet-point placeholders on the **"TABLE OF CONTENTS"** page must now be replaced with SVG transparent images. Each Table of Contents heading has a corresponding SVG image and GitHub Raw URL. Implement a **deterministic name-based SVG mapping system** so that the correct SVG image is automatically embedded beside its matching heading in the output PDF.

---

## 1. Implement Name-Based SVG Mapping

Create a **Heading → SVG URL** dictionary. Each **"TABLE OF CONTENTS"** heading must resolve to its corresponding SVG image **by heading name only**.

**Do not** map by:

* Array index
* List position
* Upload order
* Image numbering
* Manual ordering

The heading text itself must be the unique lookup key.

## 3. Runtime Workflow

For every Table of Contents entry:

```text
Heading
    ↓
Normalize Heading
    ↓
Find Matching Heading in Dictionary
    ↓
Retrieve GitHub Raw SVG URL
    ↓
Download SVG
    ↓
Validate SVG
    ↓
Cache SVG
    ↓
Render SVG
    ↓
Place SVG Beside Heading
    ↓
Render Heading Text
    ↓
Render Leader Dots
    ↓
Render Page Number
```

Every Table of Contents entry must complete this workflow successfully before the PDF is exported.

---

## 4. SVG Rendering Requirements

* Load SVGs directly from the specified GitHub Raw URLs.
* Preserve original vector quality.
* Preserve transparency.
* Preserve `viewBox`, fills, and strokes.
* Do not rasterize or convert to low-resolution images.
* Maintain sharp, scalable vector rendering in the PDF.
* Use consistent icon size for every entry.
* Maintain uniform spacing and vertical alignment between the SVG and its heading.

---

## 5. Cache Management

* Cache successfully downloaded SVG files.
* Reuse cached SVGs when valid.
* If a cached SVG is missing or invalid, automatically re-download it.
* Never cache empty or corrupted SVG files.
* Ensure stale cache entries are replaced with valid versions.

---

## 6. Error Handling

If an SVG cannot be loaded:

* Log the heading name.
* Log the corresponding GitHub URL.
* Log the download or parsing error.
* Continue rendering the remaining Table of Contents entries without interrupting PDF generation.

---

## 7. Validation Before PDF Export

Before generating the final PDF, verify that:

* Every Table of Contents heading exists in the mapping dictionary.
* Every heading resolves to the correct GitHub Raw SVG URL.
* Every SVG downloads successfully or loads from a valid cache.
* Every SVG is rendered beside its corresponding heading.
* All SVGs maintain consistent size, spacing, and alignment.
* No SVG is missing, duplicated, distorted, clipped, or misaligned.
* No bullet-point placeholders remain.
* The final **"TABLE OF CONTENTS"** page displays the correct SVG image for every heading with professional, print-ready formatting that complies with Amazon KDP standards.

### Required Fix: "MAP OF YOUR ADVENTURE" SVG Icons / Bullet Points Rendering Logic

The **"MAP OF YOUR ADVENTURE" → Bulk SVG Upload (Index)** feature is currently not functioning correctly.

As shown in the attached screenshot, regardless of whether the user selects **SVG Icons** or **Bullet Points**, only the first few entries are rendered and the remaining page is left blank. The selected mode is not being applied, causing an incomplete **MAP OF YOUR ADVENTURE** page.

Implement the following logic:

#### 1. Respect the User's Selection

* Read the user's selected display mode before rendering the page.
* If **SVG Icons** is selected, render **only the uploaded SVG icons**.
* If **Bullet Points** is selected, render **only bullet points**.
* Never mix both modes on the same page.
* The renderer must use only the currently selected mode.

---

#### 2. Render Every Index Entry

* Iterate through the complete "MAP OF YOUR ADVENTURE" data source.
* Render **every section** in order until the last entry.
* Do not stop after the first few rows.
* Verify that the rendering loop processes the full dataset before exporting the PDF.

---

#### 3. Fix the Layout Engine

Before rendering each row:

* Calculate the remaining printable height.
* If the current page has enough space:

  * Render the next row.
* Otherwise:

  * Automatically create a new "MAP OF YOUR ADVENTURE" page.
  * Continue rendering the remaining rows.
* Never stop rendering because of insufficient page space.

---

#### 4. Dynamic Pagination

Do not assume that all entries fit on one page.

Instead:

* Measure the height of each rendered row.
* Maintain consistent spacing between rows.
* When the remaining height is insufficient, automatically insert another page.
* Continue until every entry has been rendered.

---

#### 5. SVG Icon Mode

When **SVG Icons** is selected:

* Load every uploaded SVG.
* Validate that the SVG exists.
* Render the SVG at the configured icon size.
* Maintain the original vector quality.
* Preserve transparency.
* Vertically center each icon with its corresponding heading.
* Align all icons consistently.

If an SVG fails to load:

* Report which icon failed.
* Do not terminate rendering.
* Continue rendering the remaining entries.

---

#### 6. Bullet Point Mode

When **Bullet Points** is selected:

* Ignore all SVG upload mappings.
* Render the configured bullet symbol only.
* Use consistent bullet size, alignment, spacing, and typography.
* Ensure bullets align perfectly with the associated heading.

---

#### 7. Rendering Validation

Before exporting the PDF, validate that:

* The selected mode is correctly applied.
* Every "MAP OF YOUR ADVENTURE" entry has been rendered.
* No entries are skipped.
* No blank areas appear because the rendering loop terminated early.
* Icons or bullets are visible for every row.
* Leader dots and page numbers remain aligned.
* Headings are not clipped.
* Borders, margins, and typography remain consistent.

---

### Expected Result

The **"MAP OF YOUR ADVENTURE"** page should render **100% of the index entries** using the user's selected mode (**SVG Icons** or **Bullet Points**). If the content exceeds one page, additional pages must be created automatically. The final PDF should contain no missing entries, no large blank areas, no incomplete rendering, and should maintain a clean, professional layout fully compliant with Amazon KDP standards.

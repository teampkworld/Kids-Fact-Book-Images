### 2. Correct Category-to-Icon Mapping

The current icon assignment logic is incorrect. Many facts are displaying icons that belong to different categories.

#### Required Logic

* Review **every fact category** and assign an icon that accurately represents its topic.
* Do not assign icons randomly.
* The icon selection should be determined automatically based on the fact's category.

For example:

* Animals → Animal-related icon
* Space → Telescope, Planet, or Rocket icon
* Science → Microscope or Laboratory icon
* Geography → Globe or Map icon
* Human Body → Human body or Brain icon
* Ocean → Fish or Ocean icon
* Nature → Leaf or Tree icon
* History → Castle, Scroll, or Monument icon
* Technology → Computer or Robot icon
* Sports → Trophy or Sports icon

Create a centralized category-to-icon mapping table and use it throughout the application so the correct SVG icon is always selected automatically.

## Custom Presets Category

Create a new category tab:

**Gradient Presets**

* Popular
* Modern
* Pastel
* Neon
* Nature
* Dark
* Rainbow
* **Custom Presets** ← **(New Category)**

Selecting **Custom Presets** should display every palette I supplied.

---

## Palette Implementation

Each preset consists of **exactly four HEX colors**.

Example:

```
#FFDD9C
#F9B637
#FB6C00
#E73F1E
```

This should become one preset.

The next four HEX values become the next preset, and so on until all provided palettes have been imported.

Each preset should render as a live gradient preview using its four colors.

---

## Preserve the Exact Colors

Do **not** modify, normalize, replace, reorder, or interpolate the HEX values.

Requirements:

* Preserve every HEX code exactly as provided.
* Preserve the original color order.
* Preserve the intended gradient appearance.
* Do not generate alternative colors.
* Do not automatically brighten, darken, or adjust saturation.

The application must use the exact palette values supplied.

---

## Automatic Duplicate Detection

Some palettes in my list are intentionally repeated.

Implement duplicate detection during import.

* If two presets contain the exact same four HEX colors in the same order, import only one copy.
* Keep only unique palettes.
* Do not remove palettes that differ by even a single HEX value or color order.

This prevents duplicate entries while preserving all unique designs.

---

## Preset Card Design

Each preset should appear as a preview card showing:

* Live gradient preview
* Preset name (for example: **Custom 001**, **Custom 002**, **Custom 003**, ...)
* Four-color gradient representation

The preview should accurately display the final applied gradient.

---

## Scrollable Gallery

Since there are many custom presets:

* Display them in a scrollable gallery.
* Support smooth scrolling.
* Use lazy loading or virtualization if necessary for performance.
* Maintain fast rendering even with hundreds of presets.

---

## One-Click Apply

When the user selects any preset:

* Apply the gradient immediately to the SVG artwork.
* Update the preview instantly.
* Preserve SVG transparency.
* Preserve SVG vector quality.
* Do not modify SVG paths or geometry.

No manual refresh or regeneration should be required.

---

## Future Expansion

Design the **Custom Presets** category so additional palettes can be added simply by appending new HEX color sets to the preset data source.

The UI should automatically detect and display new presets without requiring changes to the application logic.

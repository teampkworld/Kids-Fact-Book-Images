2). 
IMPORTANT INDEX LAYOUT RULES

DO NOT attempt to fit the entire index onto one page.

DO NOT automatically reduce the font size.

DO NOT compress the spacing.

DO NOT overlap text.

DO NOT shrink line spacing.

DO NOT create unreadable pages.

Every index page must remain clean and comfortable to read.

--------------------------------------------------

MULTI-PAGE INDEX

Generate as many Index pages as required.

Never limit the Index to a fixed number of pages.

Continue creating new pages whenever the current page becomes full.

The Index should naturally flow across multiple pages.

Example

Page 1
A–B

Page 2
C–D

Page 3
E–G

Page 4
H–K

Page 5
L–M

Page 6
N–P

Page 7
Q–S

Page 8
T–Z

If one letter contains many entries (for example S or M), continue that same letter onto the next page.

Never force an entire alphabet section onto one page.

--------------------------------------------------

PAGE CAPACITY

Each Index page should contain approximately 45–70 entries maximum.

Never exceed the readable capacity of the page.

If more entries exist, automatically continue onto the next page.

--------------------------------------------------

COLUMN LAYOUT

Use TWO balanced columns.

Columns must have equal width.

Leave a clear gutter between columns.

Maintain generous page margins.

Never allow text to touch page edges.

--------------------------------------------------

FONT SIZE

Index title:
20–24 pt

Alphabet heading:
16–18 pt Bold

Index entries:
10–11 pt

Never use text smaller than 10 pt.

--------------------------------------------------

LINE SPACING

Maintain consistent spacing.

Minimum spacing:

1.2–1.3 line height.

Leave extra spacing before every alphabet heading.

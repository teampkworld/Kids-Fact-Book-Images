### Add Bulk SVG Icon Upload Feature

Please add a new **Bulk SVG Icons Upload** option that allows users to upload multiple **transparent SVG icons (1:1 aspect ratio)** for use throughout the book.

### Requirements

* Add a dedicated uploader for **Bulk SVG Icons** in the **Icon & Illustration Style** section.
* Support uploading multiple **transparent SVG files** at once.
* Keep the existing options **Color Icons (Default)** and **Black & White Icons** unchanged.

### Automatic Icon Assignment

After the user uploads the SVG icon pack:

* Automatically analyze the content and category of each fact.
* Detect the most appropriate SVG icon for each fact based on its title, category, and description.
* Replace the current icon displayed before each fact with the best-matching uploaded SVG icon.
* Remove all existing default icons and use only the uploaded SVG icons throughout the book.
* Each uploaded icon should be reusable and assigned wherever it best matches the fact content.

### Matching Logic

The icon selection should intelligently match facts based on their subject, such as:

* Animals
* Space
* Science
* Human Body
* Nature
* Geography
* Ocean Life
* Dinosaurs
* History
* Technology
* Weather
* Plants
* Insects
* Food
* Sports
* Music
* Art
* Transportation
* Mathematics
* and other relevant categories.

If multiple uploaded icons fit a category, automatically choose the most suitable one.

### Display Requirements

* Preserve the original SVG transparency.
* Maintain the original aspect ratio.
* Display icons with consistent sizing, alignment, spacing, and padding throughout the book.
* Render the SVGs at high quality without distortion, cropping, pixelation, or loss of detail.
* Ensure all icons print cleanly in the generated PDF.

The result should be a fully automated workflow where users can upload a single collection of transparent SVG icons, and the application intelligently assigns and embeds the most relevant icon before each fact, replacing the existing default icons while preserving the current **Color Icons (Default)** and **Black & White Icons** style options.

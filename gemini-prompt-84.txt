## Fix Book Spine Title Editor – Font Size Scaling Logic

### Current Issue

The **Font Size** control in the **Book Spine Title Editor** is not scaling correctly.

For example, when the user selects a font size of **6 pt**, the rendered spine title becomes **much larger than expected**. The displayed text size does not accurately reflect the selected point size, making the spine preview unrealistic and causing the text to overflow outside the spine area.

The attached screenshots clearly show that the rendered font size is significantly larger than the selected value.

---

## Required Fix

Correct the font size calculation so that the displayed spine title accurately matches the font size selected by the user.

For example:

* 6 pt should render as an actual **6 pt** font.
* 8 pt should render as an actual **8 pt** font.
* 10 pt should render as an actual **10 pt** font.
* Every font size should scale proportionally and consistently.

The preview and exported PDF must use the same font size calculation.

---

## Font Size Scaling Requirements

Implement proper font scaling based on the selected point size.

The rendering engine must:

* Use true typography point sizes (1 pt = 1/72 inch).
* Avoid multiplying or exaggerating the selected value.
* Ensure every font size increases gradually and proportionally.
* Prevent sudden jumps in size between adjacent values.
* Maintain consistent scaling across all supported fonts.

---

## Spine Auto-Fit Logic

After applying the correct font size:

* Automatically calculate the available spine width.
* If the title fits, render it at the selected font size.
* If the title exceeds the available spine area, reduce the font size gradually until it fits completely.
* Never allow text to overflow, clip, or extend beyond the spine boundaries.

---

## Live Preview

Whenever the user changes the font size:

* Update the spine preview instantly.
* Recalculate the text layout automatically.
* Recalculate letter spacing and line spacing if necessary.
* Keep the title centered within the spine.

No manual refresh or regeneration should be required.

---

## PDF Export

The exported KDP cover PDF must use the **exact same font size and layout** shown in the live preview.

There must be no difference between the preview and the final PDF.

---

## Validation Checklist

Before generating the final cover PDF, verify that:

* ✅ The selected font size matches the actual rendered size.
* ✅ Font sizes increase smoothly and proportionally.
* ✅ No oversized rendering occurs at small values such as **6 pt**.
* ✅ The title always fits within the spine boundaries.
* ✅ The live preview updates immediately after every font size change.
* ✅ The exported PDF matches the live preview exactly.
* ✅ The font size logic works correctly for all fonts, all book sizes, and all spine widths.

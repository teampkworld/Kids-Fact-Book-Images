Fix Inconsistent "TABLE OF CONTENTS" PDF Generation Across Different Computers

I have tested the same project on two different computers. The generated **TABLE OF CONTENTS** page is inconsistent. i attached two screen shots for reference.

* **Computer 1:** The entire Table of Contents is generated correctly.
* **Computer 2:** Only the first two entries are rendered, and the remaining page is blank.

This should **never happen**. The PDF generation must be **100% deterministic**, meaning the exact same project, data, and code must always generate an identical PDF on every computer.

---

# Required Fix

Do **not** apply a temporary workaround. Find and fix the root cause.

The PDF generation pipeline must be completely deterministic and independent of:

* Computer performance
* CPU speed
* RAM
* Internet speed
* Cache state
* Browser timing
* Operating system
* Rendering order

---

# 1. Lock the PDF Generation Pipeline

The PDF must be generated using a strictly sequential workflow.

```text
Load Book Data
        ↓
Load Fonts
        ↓
Embed Fonts
        ↓
Load SVG Icons
        ↓
Validate SVG Icons
        ↓
Cache SVG Icons
        ↓
Calculate Final Page Numbers
        ↓
Generate Complete Table of Contents
        ↓
Render Every TOC Entry
        ↓
Verify Entry Count
        ↓
Generate PDF
        ↓
Export PDF
```

No later step may begin until the previous step has completed successfully.

---

# 2. Remove Every Race Condition

Review the complete PDF generation codebase for:

* async functions
* Promise chains
* setTimeout()
* setInterval()
* requestAnimationFrame()
* useEffect timing
* lazy loading
* parallel rendering

The PDF export must **never** begin while any resource is still loading.

Every asynchronous operation must be awaited before continuing.

---

# 3. Wait Until Every Resource Is Ready

Before generating the Table of Contents, verify that **all** required resources have finished loading.

This includes:

* Fonts
* SVG icons
* Images
* Book metadata
* Page numbers
* Heading list
* Index entries

Do not continue until every resource has been successfully prepared.

---

# 4. Lock the Final Page Numbers

The page numbers used in the Table of Contents must be calculated only after the entire book has been generated.

Once calculated:

* Freeze them.
* Do not recalculate them later.
* Do not allow page shifts during rendering.

---

# 5. Prevent the Rendering Loop From Stopping

The rendering loop must never terminate because of:

* one missing SVG
* one invalid heading
* one failed image
* one rendering exception

If one entry fails:

* log the error
* render a fallback if necessary
* continue rendering every remaining entry

One failed item must never stop the entire Table of Contents.

---

# 6. Validate Every SVG

For every SVG:

* Verify HTTP 200 response.
* Verify valid `<svg>` root element.
* Verify valid viewBox.
* Verify non-empty content.
* Verify successful parsing.

If invalid:

* remove the cached version
* re-download
* validate again

Only then continue.

---

# 7. Embed Fonts Properly

Do not rely on locally installed fonts.

Embed every font directly into the PDF.

Every computer must render identical typography.

No font substitution should ever occur.

---

# 8. Standardize Measurements

Never use browser-dependent measurements.

Use fixed PDF units only.

Keep identical:

* margins
* padding
* icon size
* line height
* font size
* leader dots
* spacing

on every machine.

---

# 9. Validate Before Export

Immediately before exporting the PDF, perform these checks:

* ✓ Fonts embedded
* ✓ All SVGs loaded
* ✓ All headings generated
* ✓ Page numbers finalized
* ✓ Expected TOC entry count matches rendered entry count
* ✓ No pending async operations
* ✓ No rendering exceptions
* ✓ No missing icons
* ✓ No missing headings
* ✓ No blank TOC area

If any validation fails, **cancel PDF generation**, report the exact error, and do not export a partial or corrupted PDF.

---

# 10. Debug the Root Cause

Do not assume the implementation is correct.

Instrument the code with runtime logging.

For every Table of Contents entry, log:

* Heading name
* Dictionary lookup result
* SVG URL
* Download status
* Cache status
* SVG validation result
* Render status
* X/Y coordinates
* Render completion

Also log:

* Total expected TOC entries
* Total rendered TOC entries
* Total skipped entries
* Total failed entries

If the rendered count does not equal the expected count, identify the exact stage where processing stopped.

---

# Expected Result

The **same project**, using the **same data** and **same code**, must produce a **pixel-identical PDF** on every computer, regardless of hardware, browser, cache state, or internet speed. The **TABLE OF CONTENTS** page must always render every heading, icon, leader dot, and page number completely, with no missing entries or blank areas.

### Fact Category → SVG Icon Mapping Logic

In every fact box, the **category name appears immediately after the Fact ID** (for example, **Fact #126 — Animal**, **Fact #139 — Mathematics**, **Fact #245 — Space**, etc.). Use this category text as the primary key for selecting the correct SVG icon.

Implement the following logic:

* Read the category name displayed after the Fact ID.
* Match that category name with the uploaded SVG icon filename.
* Display the SVG icon whose filename corresponds exactly to that category.
* Do **not** determine the icon based on the fact description, upload order, file index, random selection, or cached mappings.
* The **category label shown in the fact header is the authoritative source** for icon selection.

### Validation Rules

* Extract the category directly from the fact header (for example, `Fact #126 — Animals`).
* Match the extracted category with the uploaded SVG filename.
* Ignore differences in capitalization, spaces, underscores, or hyphens when matching.
* Never assign icons randomly.
* Never assign icons by upload order or file position.
* Never infer the icon from the fact description when the category is already provided in the header.
* Every fact belonging to the same category must always display the same SVG icon.
* Apply this logic consistently in the editor, live preview, PDF proof, final manuscript, DOCX export, XLSX export, and any future export formats.
